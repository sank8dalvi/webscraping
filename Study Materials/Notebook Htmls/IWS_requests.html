Requests¶

The requests library is the de facto standard for making HTTP requests in Python. It abstracts the complexities of making requests behind a beautiful, simple API so that you can focus on interacting with services and consuming data in your application.¶

In [1]:
import requests as req
# http://docs.python-requests.org/en/master/

A simple get request¶

In [3]:
url = 'http://localhost/index2.html'
In [4]:
resp = req.get(url)

Contains important info like:

• Content-Type: type of response eg. html, json, jpeg, etc.

• Content-Length: Size of the content in bytes

• Last-Modified: last modified date and time of the page requested

File downloading¶

Files which have a large size and whose data cannot be directly assigned to a variable and downloaded in parts.

Why? If the file is too large, the system may crash trying to load it into the RAM also known as RAM leak.

We use iter_content in the response object to work with such files. By iterating through a fixed portion of the file we thus avoid potential RAM leaks.

We can also use this loop to keep the user updated about the progress of the download.

In [16]:
resp = req.get(url, stream=True)
In [17]:
with open('file.jpeg', 'wb') as fd:
    for chunk in resp.iter_content(chunk_size=128):
        fd.write(chunk)

GET request¶

GET request method retrieves information from the server. As part of a GET request, some data can be passed within the URL's query string, specifying (for example) search terms, date ranges, or other information that defines the query.

Used by all search engines to send the query string to its servers

POST request¶

POST request method requests that a web server accepts the data enclosed in the body of the request message, most likely for storing it. It is often used when uploading a file or when submitting a completed web form.

Used for sending username/email and pssword details to the server.

Other methods of requestsing content from a server¶

HEAD¶

The HEAD method asks for a response identical to that of a GET request, but without the response body. This is useful for retrieving meta-information written in response headers, without having to transport the entire content.

Uses:

• Querying for size of a file before downloading it.

• To check whether a file has changed

url = 'https://google.com'
In [32]:
resp = req.head(url)

PUT¶

The PUT method requests that the enclosed entity be stored under the supplied URI. If the URI refers to an already existing resource, it is modified; if the URI does not point to an existing resource, then the server can create the resource with that URI. method support

DELETE¶

The DELETE method deletes the specified resource. method support

OPTIONS¶

The OPTIONS method returns the HTTP methods that the server supports for the specified URL. method support

PATCH¶

The PATCH method applies partial modifications to a resource. method support

TRACE¶

The TRACE method echoes the received request so that a client can see what (if any) changes or additions have been made by intermediate servers.

CONNECT¶

The CONNECT method converts the request connection to a transparent TCP/IP tunnel, usually to facilitate SSL-encrypted communication (HTTPS) through an unencrypted HTTP proxy.
